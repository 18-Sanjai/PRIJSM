## 💡 System Architecture Design (3-Tier Separation)

Although the application compiles as a highly cohesive standalone script, the architectural framework is cleanly decoupled into three core operational tiers:

### 1. 🎨 Presentation Tier (Frontend UI Layer)
* **Visual Workspaces UI:** Responsive web layout container blocks engineered with an executive *Elite Tech Cyber-Dark Palette* (Carbon Black background, Cyber Yellow headers, and Neon Cyber Blue indicator highlights).
* **Interactive Dynamic Elements:** Standard native input form controllers tracking real-time data inputs (`st.slider` structures tracking account age in days, item selections, and price parameters text fields).
* **Analytical Metrics:** Live scoreboards displaying high-contrast aggregated statistical telemetry boxes utilizing secure inline presentation rules.

### 2. 🧠 Application Tier (Backend Processing & AI Logic)
* **The JSM Scoring Brain:** Programmed mathematical weight profiling system mimicking an optimized, fine-tuned **Gradient Boosted Tree Classifier (XGBoost logic proxy engine)** calculating absolute Risk Indices from 0% to 100%.
* **Secure SMTP Corporate Email Relay:** Fully automated communication helper node leveraging Python's native `smtplib` and `MIMEMultipart` transport layers to compress, package, and dispatch real-time CSV audit attachments directly to target auditor mailboxes in under 5 seconds via custom secure relays (`smtp.gmail.com:587`).

### 3. 🗄️ Database Tier (Data Persistence & SQL Internals)
* **ACID Relational Engine:** Runs a transactional storage memory layer initialized via native `sqlite3` driver connectors mapping strict schemas layout keys (`customer_profiles`, `return_requests_queue`, `fraud_audit_trails`).
* **On-The-Fly Auto-Registration:** Active query injection sequences that verify inbound client metadata. If a customer profile is missing during the evaluation, the backend instantly issues automated `INSERT` and `conn.commit()` statements to register the user permanently across active sessions.
* **Industrial Database Seeding Function:** Automatically populates the database structure with over **200+ diversified historic shopping entries** at application launch, ensuring all executive analytics charts map out dense telemetry from second one.

## 📊 Streamlined Application Flow (4 Easy Multi-Page Tabs)

The workspace completely eliminates dense, confusing artificial intelligence jargon, replacing them with simple, user-friendly plain English business tags organized into four dedicated operational sections:

* **PAGE 1: 📊 Sales & Return Tracker:** Continuous monitoring dashboard tracking cumulative metrics in INR (₹) like *Total Sales Value Checked*, *Fraud Prevented Value*, *Loss Ratio Indicators*, and a dense real-time incoming request logging table.
* **PAGE 2: 🧠 Check Customer Risk:** The interactive testing sandbox form mapping out client parameters linked to a dynamic SVG radial ring gauge. Enforces tier-based calibrated guardrails:
  * 🟢 **Green Gate (<30% Risk):** Payout auto-approved instantly with zero friction.
  * 🟡 **Amber Gate (30% - 70% Risk):** Hold status enforced, routing tickets to manual warehouse check queues.
  * 🔴 **Red Gate (>70% Risk):** Payout frozen, account flagged, and logged into secure fraud audit logs.
* **PAGE 3: 🗄️ Master Database History:** The multi-filter master ledger table with dynamic date sorting sorting capabilities. Features a 50-50 symmetric column panel grouping a criteria-sliced file spreadsheet extractor (**"Download Complete Audit Report as CSV"**) alongside our active direct email engine pipeline (**"Dispatch Report Data to Mail Pipeline"**).
* **PAGE 4: 📈 Simple Business Reports:** Business performance charts tracking dynamic financial timelines and category-level vulnerability scopes (safely isolating vulnerable trends like *Luxury Watches* and *Electronics* apart from protected divisions like *Standard Clothes*).
